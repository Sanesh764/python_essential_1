üí° How it Works: Conditional Execution Flow

The Python interpreter evaluates conditions sequentially. As soon as a condition is found to be True, its corresponding code block is executed, and the rest of the elif and else blocks are skipped. If no `if` or `elif` conditions are met, the `else` block (if present) is executed.

üìä Practical Example: Sum Calculator with For Loop

# Calculate sum of numbers from 1 to n
try:
    n = int(input("Enter a positive integer: "))
    if n <= 0:
        print("Please enter a positive integer.")
    else:
        sum_total = 0
        print(f"Calculating sum from 1 to {n}:")
        for i in range(1, n + 1):
            sum_total += i # Adds current number to sum_total
            print(f"Adding {i}, current sum: {sum_total}")

        print(f"\nFinal sum of 1 to {n}: {sum_total}")

        # Verification using mathematical formula (Gaussian sum)
        formula_sum = n * (n + 1) // 2
        print(f"Verification using formula: {formula_sum}")
except ValueError:
    print("Invalid input. Please enter a whole number.")

üí° How it Works: For Loop Iteration

A for loop iterates over (goes through) each item in a sequence (like a list, string, or `range` object) one by one. In each iteration, the loop variable (e.g., `i` or `letter`) takes on the value of the current item, and the code inside the loop's block is executed.

‚ö†Ô∏è Important: Avoiding Infinite Loops

A `while` loop continues executing its code block as long as its condition remains True. It's crucial to include code inside the loop that will eventually make the condition False, otherwise, the loop will run forever (an "infinite loop"), consuming resources and freezing your program.

Error Handling - Making Programs Robust

üö´ Understanding Errors in Python

  • Syntax Errors: Occur when the Python interpreter cannot understand the code due to incorrect grammar (e.g., misspelled keywords, missing colons). These prevent the program from running.
  • Runtime Errors (Exceptions): Occur during program execution, even if the syntax is correct (e.g., trying to divide by zero, accessing a list index that doesn't exist, trying to convert text to an integer).
  • Logical Errors: The program runs without crashing, but it produces incorrect or unexpected results because of flaws in the program's logic. These are the hardest to debug!

üö® Example of a Runtime Error: ZeroDivisionError

# This code will cause a ZeroDivisionError
numerator = 10
denominator = 0
# result = numerator / denominator # Uncommenting this line will cause an error
print("This line will not be reached if an error occurs above.")

When Python tries to execute `10 / 0`, it raises a `ZeroDivisionError`, and the program terminates abruptly.

Error Handling with `try-except` Block

üõ°Ô∏è The `try-except` Mechanism

The `try-except` block allows you to "try" to execute a block of code and "catch" (handle) specific errors (exceptions) that might occur. This prevents your program from crashing and allows you to provide a graceful response to the user.

# Handling a potential ZeroDivisionError
try:
    num1 = int(input("Enter numerator: "))
    num2 = int(input("Enter denominator: "))
    
    result = num1 / num2
    print(f"Result of division: {result}")
except ZeroDivisionError:
    print("Error: Cannot divide by zero!")
except ValueError:
    print("Error: Invalid input. Please enter only numbers.")
except Exception as e: # Catch any other unexpected errors
    print(f"An unexpected error occurred: {e}")
finally:
    print("Division attempt complete.") # This line always executes

üí° How it Works: `try-except` Flow

  • The code inside the **`try`** block is executed first.
  • If an error (exception) occurs in the `try` block, Python immediately jumps to the **`except`** block that matches the type of error. The code inside the matching `except` block is then executed.
  • If no error occurs in the `try` block, the `except` blocks are skipped.
  • The **`else`** block (optional) is executed if the `try` block completes successfully (i.e., no exception occurred).
  • The **`finally`** block (optional) always executes, regardless of whether an exception occurred or was handled. It's often used for cleanup operations (e.g., closing files).

# Basic IP Address Validator Function (Conceptual)
def is_valid_ipv4(ip_address):
    try:
        parts = ip_address.split('.')
        if len(parts) != 4:
            return False
        for part in parts:
            num = int(part)
            if not (0 <= num <= 255):
                return False
        return True
    except ValueError:
        return False

print(f"Is '192.168.1.1' valid? {is_valid_ipv4('192.168.1.1')}") # True
print(f"Is '256.0.0.1' valid? {is_valid_ipv4('256.0.0.1')}")   # False
print(f"Is '1.2.3' valid? {is_valid_ipv4('1.2.3')}")         # False
print(f"Is 'abc.def.ghi.jkl' valid? {is_valid_ipv4('abc.def.ghi.jkl')}") # False
